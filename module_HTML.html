HTML | La Structure d'un document HTML

📙 HTML = des boîtes qui contiennent d’autres boîtes

Imaginons que chaque balise HTML est une boîte. Certaines boîtes peuvent en contenir d’autres, comme des poupées russes.
Voici la structure générale : 

<!DOCTYPE html>
<html lang="fr">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mon document HTML</title>
    <link rel="stylesheet" href="./css/styles.css">
    <script src="./js/script.js" defer></script>
  </head>

  <body>
    Le contenu principal de mon document HTML.
  </body>

</html>
  • <!DOCTYPE html> → indique que le document est en HTML5.
  • <html> → la grande boîte qui contient tout.
  • <head> → boîte invisible (réglages, titre, CSS…).
  • <body> → boîte visible (tout ce que voit l’utilisateur).
Version ultra visuelle 
<!DOCTYPE html>   ← Étiquette : “Ceci est un document HTML5”
<html>            ← Grande boîte principale
  <head>          ← Boîte invisible
  </head>
  <body>          ← Boîte visible
  </body>
</html>

📙 Les principales boîtes (balises) à connaître

  1. Les boîtes de texte :
    2. <h1> à <h6> → titres (du plus important au moins important)
    <p> → paragraphe
    <span> → petit morceau de texte dans une phrase
    <strong> → texte important
    <em> → texte mis en avant (italique)

  2. Les boîtes de structure :
    3. <header> → en-tête
    <nav> → menu
    <main> → contenu principal
    <section> → grande section
    <article> → contenu indépendant
    <footer> → bas de page
    <div> → boîte neutre pour organiser

  3. Les boîtes de contenu :
    4. <img> → image
    <a> → lien
    <ul> / <ol> / <li> → listes
    <video> → vidéo
    <form> → formulaire

📙 Quand utiliser quelle boîte ?

Une règle simple : si le contenu a un sens précis, on utilise une balise sémantique.
Exemples :

un menu → <nav>
un article de blog → <article>
une zone de contenu → <section>
un pied de page → <footer>

Si on veut juste organiser visuellement, on utilise <div>.

HTML : Balises Structurantes

Les balises structurantes en HTML moderne servent à donner du sens à notre code : elles indiquent clairement quelles parties d’une page jouent quels rôles, ce qui améliore l’accessibilité, le référencement et la lisibilité du code.

📙 Une balise structurante décrit la fonction d’une zone de la page

Par exemple :

<main> = le contenu principal, unique sur la page
<section> = une partie thématique du contenu
<article> = un contenu autonome (post, article de blog…)
<nav> = une zone de navigation
<footer> = le bas de page (infos légales, liens…)

⚠️ Important : on ne met pas des <div> partout. On choisit une balise qui explique le rôle de la zone.

Ça aide :

  • les lecteurs d’écran
  • les moteurs de recherche
  • les développeurs qui lisent ton code
  • toi-même
Liste des principales balises structurantes :
<header> zone d'en-tête : logo, titre, navigation principale
<nav> zone dédiée aux liens de navigation
<main> contenu principal de la page (unique)
<section> partie thématique du contenu
<article> contenu autonome (post, fiche produit…)
<aside> contenu secondaire (encadrés, pubs, infos connexes)
<footer> pied de page : mentions légales, liens secondaires, copyright

HTML : Les balises textuelles

Les balises textuelles permettent de structurer le texte et de donner du sens aux mots, phrases et paragraphes. Elles indiquent ce que représente un morceau de texte : un titre, un paragraphe, une citation, un mot important, etc.
Elles ne servent pas à la mise en forme (ça, c’est le rôle du CSS), mais à la sémantique.

Ça aide :

  • Les lecteurs d’écran comprennent mieux la page
  • Les moteurs de recherche savent ce qui est important
  • Le code est plus clair pour nous et les autres
  • Le style CSS devient plus facile à gérer

📙 Les grandes catégories de balises textuelles

  1. Titres
    • <h1> à <h6> : titres hiérarchiques
    • Comme les titres et sous‑titres d’un livre.

  2. Paragraphes
    • <p> : bloc de texte standard

  3. Mise en valeur sémantique
    • <strong> : importance forte
    • <em> : mise en avant, intonation
    • <mark> : surlignage sémantique
    • <small> : texte secondaire
    • <abbr> : abréviation avec signification
    • <cite> : titre d'œuvre
    • <time> : date ou heure

  4. Citations
    • <blockquote> : citation longue
    • <q> : citation courte dans une phrase

  5. Listes
    • <ul> : liste non ordonnée (liste à puces)
    • <ol> : liste ordonnée (numérotée)
    • <li> : élément de liste

  6. Texte technique
    • <code> : code informatique
    • <pre> : texte préformaté
    • <kbd> : touche de clavier
    • <samp> : sortie de programme

HTML : Les balises multimédias

Les balises multimédias HTML servent à afficher des images, lire de l’audio, montrer des vidéos ou intégrer des contenus externes directement dans une page web. Elles enrichissent l’expérience utilisateur et sont aujourd’hui entièrement prises en charge par les navigateurs modernes.

📙 Pourquoi les balises multimédias existent ?

Elles permettent d’intégrer images, sons, vidéos et contenus interactifs sans plugin externe (Flash n’est plus utilisé).
Elles rendent une page vivante, informative et accessible.

📙 Les principales balises multimédias HTML

  • <img> → afficher une image
    • Affiche une image statique (photo, icône, illustration).
    • Attributs importants : src, alt, width, height.

  • <audio> → lire un son
    • Permet de lire un fichier audio (musique, voix, bruitage).
    • Peut afficher des contrôles : play, pause, volume.
    • Supporte plusieurs formats via <source>.
      • Exemple : 
      <audio controls>
  <source src="musique.mp3" type="audio/mpeg">
  <source src="musique.ogg" type="audio/ogg">
</audio>

  • Formats audio / vidéo recommandés
    • Vidéo recommandée : MP4 (format universel)
    • Audio recommandé : MP3 (musique compressée)

  • <track> → sous-titres et descriptions
    • Ajoute des sous-titres, descriptions audio, traductions.
    • S’utilise à l’intérieur de <video>.

  • <map> + <area> → zones cliquables sur une image
    • Transforme une image en carte interactive.

  • <object> et <embed> → contenus externes
    • Intègrent des PDF, animations, fichiers divers.
    • <embed> est plus simple et direct.

HTML : balise <a> (lien hypertexte)

C’est la balise de navigation principale du web. Elle sert à créer un lien vers :

  • une autre page
  • un fichier
  • une ancre interne
  • un email
  • un numéro de téléphone

📌 Toutes les variantes de liens (internes, externes, ancres) utilisent la même balise <a>. Ce qui change, c’est le type d’URL dans l’attribut href.

📙 Les liens externes

Ce sont des liens qui pointent vers un autre site, en dehors de ton domaine.

  • Syntaxe :
<a href="https://www.google.com">Google</a>
  • Caractéristiques :
    • L’URL est absolue (commence par https://).
    • Souvent on ajoute :
target="_blank" rel="noopener noreferrer"

→ Pour ouvrir dans un nouvel onglet et améliorer la sécurité.

📙 Les liens internes

Ce sont des liens qui pointent vers une autre page du même site ou vers une section de la même page.

  • Lien interne vers une autre page du site :
<a href="contact.html">Contact</a>

Ici, l’URL est relative (pas de https://).

  • Lien interne vers une section de la même page (ancre) :
  1. On place un id sur la cible :
<h2 id="services">Nos services</h2>
    start = "2"
  1. On crée le lien :
<a href="#services">Aller aux services</a>
  • Lien interne vers une section d’une autre page :
<a href="about.html#team">Notre équipe</a>

HTML : Balises fonctionnelles

Les balises <link>, <script> et <style> ne sont pas des balises de contenu : ce sont des balises fonctionnelles placées dans le <head>. Elles servent à charger des ressources externes ou du code. Elles ne produisent rien à l’écran, mais indiquent au navigateur quoi charger et comment traiter la page.

📙 <link> | Lien vers une ressource externe

  • Relie le document HTML à un fichier externe : CSS, icône, police, préchargement…
  • C’est une balise vide (pas de contenu).
  • Très utilisée pour charger le CSS.
Exemple 
<link rel="stylesheet" href="style.css">

Type : balise de ressource externe (metadata). Elle définit une relation entre la page et un fichier.

📙 <script> | Balise de script

  • Intègre du JavaScript, soit directement, soit via un fichier externe.
  • Peut être placée dans <head> ou <body>.
  • Peut utiliser src, async, defer.
Exemple 
<script src="app.js" defer></script>

Type : balise de script / exécution. Elle ajoute du comportement dynamique à la page.

📙 <style> | Balise de style interne

  • Permet d’écrire du CSS directement dans la page.
  • Utilisée pour des styles rapides ou spécifiques.
Exemple 
<style>body { background: lightblue; }</style>

Type : balise de style interne (CSS). Elle contient du code CSS, pas du contenu visuel.

HTML : Formulaires

📙 La Structure de base

📙 Les types de champs les plus utilisés

Les types de champs HTML les plus utilisés en pratique couvrent 95% des besoins réels : texte, email, mot de passe, nombre, choix (checkbox/radio), fichier et bouton d’envoi. Ce sont ceux que l’on retrouve dans presque tous les formulaires modernes, car ils sont simples, fiables et bien supportés par tous les navigateurs.

  1. text — Le champ universel
    • Sert à saisir du texte simple : nom, prénom, adresse…
    • C’est le type par défaut.
    • Le plus utilisé de tous.

  2. password — Mot de passe
    • Masque les caractères.
    • Utilisé dans tous les formulaires de connexion.

  3. email — Adresse email
    • Vérifie automatiquement la forme d’un email.
    • Affiche un clavier optimisé sur mobile.
    • Très utilisé dans les formulaires d’inscription.

  4. number — Nombre
    • Permet de saisir un nombre avec flèches.
    • Supporte min, max, step.
    • Très utilisé pour quantités, âges, prix.

  5. checkbox — Choix multiples
    • Pour accepter des conditions, sélectionner plusieurs options.
    • Très utilisé pour préférences, filtres, options cumulables.

  6. file — Import de fichier
    • Pour envoyer une image, un PDF, un document.
    • Très utilisé dans les formulaires administratifs ou RH.

  7. textarea — Zone de texte longue
    • Pour saisir un message, un commentaire, une description.
    • Indispensable dans les formulaires de contact.

  8. submit — Bouton d’envoi
    • Déclenche l’envoi du formulaire.
    • Sans lui, rien ne part.
