CSS | Cascading Style Sheets

Si le HTML est le squelette d'une page web (sa structure), le CSS (Cascading Style Sheets) représente son apparence et sa disposition. Son rôle unique est de dire au navigateur comment afficher les éléments. Pour un cerveau qui aime la logique et la catégorisation, l'immense liste des propriétés CSS peut sembler chaotique. En réalité, tout repose sur trois grands piliers indispensables :

  1. Les Sélecteurs : C'est le système de ciblage. Il faut savoir comment pointer exactement le bon élément HTML (un bouton spécifique, tous les paragraphes d'une section) pour le modifier, sans affecter le reste de la page.
  2. Le Modèle de Boîte (Box Model) : C'est la physique du CSS. Absolument chaque élément sur la page est une boîte rectangulaire invisible. Comprendre comment l'espace fonctionne à l'intérieur et à l'extérieur de cette boîte (marges, bordures, remplissage) est crucial pour éviter que les éléments ne se chevauchent.
  3. La Mise en Page (Layout) : C'est l'architecture. Une fois qu'on a nos boîtes, il faut savoir comment les aligner, les centrer ou les mettre en colonnes. Cela passe principalement par la maîtrise d'un outil intégré appelé Flexbox.

CSS : Les Sélecteurs

Les sélecteurs fonctionnent exactement comme un système de requêtes dans une base de données : on définit des critères pour cibler très exactement ce que l'on veut modifier sans toucher au reste.

📙 Les types de sélecteurs

    Il existe trois niveaux de ciblage fondamentaux à maîtriser absolument, du plus général au plus spécifique :
  • Le Sélecteur de Balise (La catégorie) : Il cible tous les éléments HTML d'un même type sur la page.
    • Exemple : p { color: blue; } modifiera le texte de tous les paragraphes.
    • Logique : "Applique ceci à toutes les voitures."
  • Le Sélecteur de Classe (Le groupe) : Il cible un groupe d'éléments spécifiques auxquels on a donné un nom de classe, peu importe leur balise HTML. Il s'écrit toujours avec un point (.) devant.
    • Exemple : .texte-important { font-weight: bold; }
    • Logique : "Applique ceci à toutes les voitures de sport." (On peux classer plusieurs éléments différents dans ce groupe).
  • Le Sélecteur d'Identifiant (L'élément unique) : Il cible un et un seul élément bien précis sur toute la page. Il s'écrit avec un hashtag (#) devant.
    • Exemple : #en-tete-principal { background: black; }
    • Logique : "Applique ceci à la voiture qui a cette plaque d'immatriculation précise." (Cet ID ne peut être utilisé qu'une seule fois dans ton HTML).

📙 Le Modèle de Boite

Il faut intégrer cette règle physique absolue en CSS : absolument chaque élément est une boîte rectangulaire. Même un texte qui semble avoir une forme complexe ou un simple lien, le navigateur le calcule comme un rectangle délimité.

    Cette boîte est composée de son contenu au centre, entouré de trois couches d'espace que tu peux manipuler :
  1. Le Padding (l'espace interne de la boite). Le padding crée l'espace à l'intérieur de la bordure, ce qui permet au texte de respirer sans agrandir l'objet lui-même.
  2. La Border (la bordure, qui peut être visible ou invisible)
  3. La Margin (la marge ewterne, la distance pour repousser les autres éléments de la page)

📙 La Mise en Page

L'outil incontournable pour structurer et aligner des éléments s'appelle Flexbox.
L'idée de base est très logique : on prend un élément parent (un grand conteneur) et on lui donnes le pouvoir de distribuer intelligemment l'espace entre ses enfants (les éléments situés à l'intérieur).

Imaginons un conteneur qui regroupe trois petits blocs. Par défaut, le navigateur les place de façon très basique. Pour débloquer les capacités de Flexbox (comme, par exemple, les centrer parfaitement sur un axe ou les espacer uniformément), il faut déclencher ce mode sur le conteneur parent.
la valeur pour activer ce mode est flex. La règle exacte à appliquer au conteneur parent est donc display: flex;. Une fois le mode Flexbox activé, les éléments enfants se placent par défaut en ligne au début du conteneur (sur l'axe horizontal). C'est à ce moment-là que l'on utilise des propriétés spécifiques pour dicter leur position.

Quelle serait la règle complète pour que nos petits blocs se placent parfaitement au centre ? display: justify-center;>
