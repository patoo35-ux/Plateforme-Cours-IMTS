Le CSS 3

CSS | Cascading Style Sheets

Si le HTML est le squelette d'une page web (sa structure), le CSS (Cascading Style Sheets) est le langage qui permet de donner du style à une page web. C'est lui qui transforme une structure brute en une interface agréable à regarder.

  1. Les Sélecteurs : C'est le système de ciblage. Il faut savoir comment pointer exactement le bon élément HTML (un bouton spécifique, tous les paragraphes d'une section) pour le modifier, sans affecter le reste de la page.
  2. Le Modèle de Boîte (Box Model) : C'est la physique du CSS. Absolument chaque élément sur la page est une boîte rectangulaire invisible. Comprendre comment l'espace fonctionne à l'intérieur et à l'extérieur de cette boîte (marges, bordures, remplissage) est crucial pour éviter que les éléments ne se chevauchent.
  3. La Mise en Page (Layout) : C'est l'architecture. Une fois qu'on a nos boîtes, il faut savoir comment les aligner, les centrer ou les mettre en colonnes. Cela passe principalement par la maîtrise d'un outil intégré appelé Flexbox.

CSS | Structure & Syntaxe

📙 Anatomie d'une règle CSS

Une règle CSS est un peu comme une phrase qui donne un ordre au navigateur pour changer l'apparence d'un élément précis. On peut décomposer une règle en trois parties principales :

  • Le Sélecteur : C'est le "qui". Il indique quel élément HTML tu veux modifier (par exemple : tous les titres h1, ou tous les paragraphes p).
  • La Propriété : C'est le "quoi". C'est l'aspect que tu veux changer (la couleur, la taille de la police, la largeur, etc.).
  • La Valeur :C'est le "combien" ou "comment". C'est le réglage spécifique que tu appliques à la propriété (par exemple : red, 20px, center).
              h1 { 
                color : blue;
              }
  • h1 est le sélecteur : il cible l'élément HTML (le titre de niveau 1).
  • colorest la propriété : c'est l'aspect technique que l'on veut modifier (la couleur du texte).
  • blue est la valeur
    • : c'est le réglage que l'on veut donner (texte de couleur bleue); 
              p {
                font-size : 16px;
              }
Composant Mot dans le code Rôle
Sélecteur p Cible tous les paragraphes.
Propriété font-size Définit l'aspect à changer (la taille).
Valeur 16px Donne la mesure précise.

Il y a deux petits détails de ponctuation très importants en CSS pour que le navigateur comprenne bien la règle.

  • Les deux-points (:) : Ils font le lien entre la propriété et sa valeur. C'est comme dire « égale » ou « vaut ».
    • Exemple : color : blue (la couleur vaut bleu).
  • Le point-virgule (;) : C'est le plus important ! Il indique la fin d'une instruction. Il permet de dire au navigateur : « J'ai fini de te donner cet ordre, tu peux passer au suivant ».
            p {
              color: blue;       /* Fin de la première instruction */
              font-size: 16px;   /* Fin de la deuxième instruction */
            }

⚠️ Si on oublie un seul point-virgule, le navigateur risque d'ignorer toutes les règles qui suivent. C'est l'erreur numéro 1 en CSS !

📙 Cibler des éléments plus précis avec les classes et les IDs

  • La Classe (.) : C'est comme une étiquette
    • que l'on peut coller sur plusieurs éléments. Si on veut que trois paragraphes différents soient en gras et en vert, on leur donne la même classe.
  • L'ID (#) : C'est comme un numéro de sécurité sociale. Il doit être unique dans toute la page. On l'utilise pour cibler un élément bien précis et un seul (comme un menu de navigation ou un logo).
                /* Une classe commence par un point */
                .texte-important {
                    color: red;
                }

                /* Un ID commence par un hashtag */
                #mon-logo {
                    width: 200px;
                }

On distingue les deux au niveau de la syntaxe (l'écriture du code) :

Type Signe en CSS Attribut en HTML Usage
Classe . (point) class="nom" Pour un groupe d'éléments
ID # (dièse) id="nom" Pour un élément unique
Spécificité = Priorité Dans notre fichier HTML, on a pour une liste de courses : 
            <p class="urgent">Pommes</p>
            <p class="urgent">Pain</p>
Et dans notre fichier CSS, pour les mettre en rouge : 
                .urgent {
                  color : red;
                }
Si on a une règle pour tous les paragraphes : 
                p {
                  color : blue;
                }
"Pommes" et "Pain" serant toujours de couleur rouge. .urgent gagne sur la balise p car elle est plus précise.

C'est ce qu'on appelle la spécificité (ou la priorité).

Le navigateur suit une hiérarchie pour savoir quelle règle appliquer quand elles se contredisent

  • L'ID (#) est le plus fort (le "super-chef").
  • La Classe (.) est moyennement forte (le "chef").
  • La Balise (p, h1, etc.) est la plus faible (le "soldat").

Exercice

Regarde ces morceaux de codes : 

          <! -- HTML -- !>
          <p id = "alerte-speciale" class = "info" >Attention !</p>

          /* CSS */
          p { 
            color : green;
          }

          .info {
            color : orange;
          }

          #alerte-speciale {
            color: purple;
          }
De quel couleur sera le texte ? Le texte s'affichera en violet (purple).

Le navigateur a suivi l'ordre de priorité que nous avons vu : l'ID (#alerte-speciale) a "écrasé" la classe (.info), qui elle-même avait "écrasé" la balise (p).
C'est un concept essentiel car, dans des projets réels, on se demande souvent pourquoi une couleur ne change pas comme on le veut : c'est presque toujours une question de priorité !

📙 Le Modèle de Boîte (Box Model)

En CSS, on considère que chaque élément HTML est une boîte rectangulaire.

Imaginons qu'on reçoit un tableau fragile par la poste.
Pour comprendre comment la place est occupée, on regarde quatre couches, de l'intérieur vers l'extérieur : <0l>

  • Le Contenu (Content) : C'est le texte ou l'image (le tableau lui-même).
  • Le Rembourrage (Padding) : C'est l'espace vide à l'intérieur de la boîte, entre le texte et le cadre (comme du papier bulle).
  • La Bordure (Border) : C'est le cadre de la boîte (le carton).
  • La Marge (Margin) : C'est l'espace vide à l'extérieur de la boîte, pour l'éloigner des autres boîtes voisines.

    • Si on augmente le padding, l'espace entre la bordure et le texte va s'agrandir.

    C'est exactement comme ajouter des couches de papier bulle à l'intérieur d'un colis : l'objet (le contenu) ne change pas de taille, mais la boîte devient plus "moelleuse" et l'espace entre l'objet et les parois du carton augmente.

    Astuce pour ne plus jamais confondre Padding et Margin :
    • Padding = Confort intérieur (l'espace pour que le texte "respire").
    • Margin = Distance extérieure (pour ne pas toucher les voisins).

    On utilise la margin car on veut agir sur l'espace extérieur à la boîte pour repousser les éléments voisins.

    💡 En CSS, on peut régler les quatre côtés d'un coup (Haut, Droite, Bas, Gauche) dans le sens des aiguilles d'une montre. 

                CSS : 
                  button {
                    margin: 10px 20px 10px 20px; 
                    /* Haut: 10px | Droite: 20px | Bas: 10px | Gauche: 20px */
                  }

    CSS | Les Sélecteurs à connaitre

    Voici les sélecteurs "indispensables" qu'on rencontrera '90% du temps, classés par type :

    📙 Les Sélecteurs de Base

    Ils ciblent directement ce qui est écrit dans le HTML.

    • Universel (*) : Cible tous les éléments de la page sans exception. On l'utilise souvent pour "réinitialiser" les marges par défaut.
    • De Type (h1, p, div) : Cible toutes les balises d'un certain nom.
    • De Classe (.maClasse) : Cible tous les éléments ayant cet attribut class.
    • D' ID (#monID) : Cible l'unique élément ayant cet id.

    📙 Les Sélecteurs de Relation (combinateurs)

    Ils sont très utiles pour cibler un élément selon sa place dans la structure. Imaginons cela comme un arbre généalogique.

    • Descendant (espace) : Cible un élément à l'intérieur d'un autre.
      Exemple : div p cible tous les paragraphes qui sont dans un class.
    • Enfant direct (>) :Cible uniquement les enfants directs (pas les petits-enfants).
      Exemple : ul > li cible les puces directement dans la liste.

    📙 Les Pseudo-Classes (Etats)

    Ils permettent de styliser un élément selon son état ou son comportement. Ils commencent toujours par un seul deux-points (:).

    • :hover : S'active quand la souris passe sur l'élément (très utilisé pour les boutons).
    • :active : S'active au moment précis où l'on clique.
    • :focus : S'active quand l'élément est sélectionné (par exemple, quand on clique dans un champ de formulaire).
    • :nth-child(n) : Permet de cibler, par exemple, un élément sur deux dans une liste (très pratique pour faire des tableaux à lignes alternées).

    📙 Tableau Récapitulatif :

    Sélecteur Syntaxe Usage courant
    Universel * { ... } Reset (marges à zéro)
    Type a { ... } Styliser tous les liens
    Classe .btn { ... } Créer un style de bouton réutilisable
    ID #main-nav { ... } Styliser la barre de navigation
    Survol button:hover Changer la couleur au passage de la souris
    Descendant nav a Liens situés uniquement dans le menu

    | BONNE PRATIQUE | Quand on écrit nos classes ou nos IDs, éviter les espaces et les accents. Utiliser plutôt des tirets (comme .mon-titre-bleu), c'est ce qu'on appelle le "kebab-case". C'est beaucoup plus lisible pour le code !

    CSS | Mise en forme des Tableaux

    La mise en page des tableaux en CSS est un peu particulière. Par défaut, un tableau HTML est très "brut" et serré.

    Pour bien comprendre, il faut voir le tableau comme un emboîtement : le tableau (table) contient des lignes (tr), qui contiennent des cellules de titre (th) ou de données (td).

    📙 Les Propriétés des Tableaux

    Voici les propriétés essentielles pour transformer un tableau austère en quelque chose de pro et lisible :

    1. La fusion des bordures (border-collapse)

      C'est la règle indispensable. Par défaut, le navigateur dessine une bordure autour de chaque cellule ET autour du tableau, ce qui crée un double trait assez laid.

      border-collapse: collapse; : Fusionne les doubles bordures en une seule ligne nette.

    2. L'espacement intérieur (padding)

      Comme avec le "Box Model", les cellules ont besoin de respirer. On applique le padding sur les th et les td, jamais sur le table lui-même. 

                      td, th {
                  padding: 12px;            /* Donne de l'espace autour du texte */
                  text-align: left;         /* Aligne le texte à gauche */
                }
    3. Les lignes zébrées (:nth-child)

      Pour faciliter la lecture de grandes listes (très utile pour aider l'œil à ne pas sauter de ligne, ce qui est super pour le confort visuel neuroatypique !), on utilise une pseudo-classe.

      tr:nth-child(even) Cible toutes les lignes paires.
      tr:nth-child(odd) Cible toutes les lignes impaires. 
                        tr:nth-child(even) {
                    background-color: #f2f2f2;    /* Gris très clair pour une ligne sur deux */
                  }

    📙 Un Bloc de Code "Modèle

          table {
        width: 100%;                  /* Le tableau prend toute la largeur */
        border-collapse: collapse;    /* Fusionne les bordures */
        margin: 20px 0;               /* Espace extérieur haut/bas */
      }

      th, td {
        border: 1px solid #ddd;       /* Bordure fine grise */
        padding: 12px;                /* Respiration intérieure */
      }

      th {
        background-color: #4CAF50;    /* Couleur de fond pour l'entête */
        color: white;                 /* Texte de l'entête en blanc */
      }

      tr:hover {
        background-color: #f5f5f5;    /* La ligne s'illumine au survol de la souris */
      }

    Petit détail qui tue !
    Remarque la dernière règle : tr:hover. Elle permet de mettre en évidence la ligne que l'utilisateur est en train de lire.

    CSS | Les FlexBox

    Flexbox est un système pensé spécialement pour ranger, aligner et répartir l'espace entre des éléments dans un conteneur, même si leur taille est inconnue ou dynamique.

    📙 Modèle de FlexBox

    1. La règle d'or : Le Parent et les Enfants

      Flexbox fonctionne toujours sur un système de relation directe.

      • On doit déclarer display: flex; sur la boîte parente (le conteneur).
      • Automatiquement, tous les éléments directement à l'intérieur (les enfants) deviennent des "éléments flex" et obéissent aux nouvelles règles d'alignement.
    2. Les deux axes (L'autoroute de Flexbox)

      Quand on active Flexbox, on crée deux axes invisibles :

      • L'axe principal (Main Axis) : C'est la direction dans laquelle tes éléments vont s' aligner par défaut (en ligne, de gauche à droite). On le contrôle avec flex-direction.
      • L'axe secondaire (Cross Axis) : C'est l'axe perpendiculaire (de haut en bas).
    3. Les 3 propriétés indispensables

      Une fois qu'on a écrit display: flex; sur ton conteneur, voici les trois "boutons de réglage" les plus souvent utilisés :

      Propriété Ce qu'elle fait Valeurs les plus courantes
      flex-direction Définit le sens de la file d'attente (l'axe principal). row (en ligne - par défaut), column (en colonne).
      justify-contentA ligne les éléments sur l'axe principal (les espacer sur la ligne). center (au milieu), space-between (écarter aux extrémités), flex-start (au début).
      align-items Aligne les éléments sur l'axe secondaire (les centrer verticalement). center(centré), stretch (étiré pour remplir la hauteur), flex-start (en haut).

    📙 Le retour à la ligne (flex-wrap)

    Pour compléter la méthode Flexbox, le retour à la ligne (flex-wrap), est le dernier concept très utile pour le "responsive" (faire en sorte que le site s'adapte aux téléphones) :

      Par défaut, Flexbox essaie de faire tenir tous les éléments sur une seule ligne, même s'ils doivent s'écraser.
    • flex-wrap: nowrap; (Par défaut) : Tout le monde reste sur la même ligne, quitte à déborder.
    • flex-wrap: wrap; : Si les éléments n'ont plus de place, ils passent automatiquement à la ligne suivante. C'est parfait pour une galerie d'images ou une liste de cartes.

    📙 L'espace entre les éléments (gap)

      Plus besoin de s'embêter avec des margin-right compliquées sur chaque enfant !
    • gap: 20px; : On l'écrit sur le parent, et cela crée un espace régulier de 20px entre tous les éléments (mais pas sur les bords extérieurs). C'est beaucoup plus propre.

    📙 Le Partage de l'Espace (flex-grow)

    Pour bien visualiser ce concept, il faut changer de perspective. Jusqu'à présent, on a vu comment ranger les boîtes. Avec flex-grow, on va voir comment elles se partagent l'espace vide.

      Le concept : "Le partage du gâteau"
    • Notre conteneur Flexbox est une boîte avec trois éléments à l'intérieur. Si ces trois éléments sont petits, il reste de l'espace vide à droite.
    • La propriété flex-grow (littéralement "capacité à grandir") est une consigne que l'on donne aux enfants (pas au parent !) pour leur dire comment ils doivent se goinfrer de cet espace vide restant.

    a règle mathématique (très simple !)

      flex-grow prend comme valeur un nombre entier sans unité (0, 1, 2, 3...).
    • flex-grow: 0; (C'est le réglage par défaut). L'élément dit : "Non merci, je garde ma taille de base, je ne veux pas d'espace en plus."
    • flex-grow: 1; L'élément dit : "Je prends une part de l'espace vide."
    • flex-grow: 2; L'élément dit : "Je prends deux parts de l'espace vide !"

    Si on a deux boîtes et qu'on donne flex-grow: 1 à la première et flex-grow: 2 à la deuxième, le navigateur va couper l'espace vide en 3 parts égales. Il en donnera une à la première boîte, et deux à la deuxième.

    ⚠️flex-grow ne multiplie pas la taille totale de l'élément, il répartit uniquement l'espace vide restant. Si une boîte contient un mot très long et l'autre un mot très court, même avec code>flex-grow: 1 sur les deux, elles n'auront pas exactement la même largeur finale, car elles n'avaient pas la même taille au départ !

    📙 La Compression de l'Espace (flex-shrink)

    flex-shrink suit exactement la même logique que flex-grow, mais à l'envers.

      Le concept : "S'écraser dans le métro"
    • Imaginons que notre conteneur Flexbox est une rame de métro aux heures de pointe. Les éléments à l'intérieur ont une certaine taille de base, mais soudain, il n'y a plus assez de place pour tout le monde (le conteneur est trop petit).
    • La propriété flex-shrink (littéralement "capacité à rétrécir") est une consigne donnée aux enfants pour leur dire comment ils doivent se compresser pour éviter de déborder de la rame.
      La règle mathématique (le jumeau inversé)
    • Contrairement à flex-grow (qui est à 0 par défaut), la valeur par défaut de flex-shrink est 1. C'est pour cela que les éléments Flexbox ont naturellement tendance à s'écraser gentiment pour rester sur une seule ligne.
      • flex-shrink: 1; (Par défaut) : L'élément dit : "D'accord, je vais me serrer proportionnellement avec mes voisins pour que tout rentre."
      • flex-shrink: 0; : L'élément dit : "Hors de question que je me serre !" (Il garde sa taille d'origine à 100%, quitte à déborder hors de la page ou à écraser complètement ses voisins).
      • flex-shrink: 2;: L'élément dit : "Je suis très souple, je vais me compresser deux fois plus vite que les autres."

    📙 La Taille Idéale de l'élément (flex-basis)

      flex-basis, c'est la taille idéale que l'élément aimerait avoir avant que l'on applique le partage de l'espace (flex-grow) ou la compression (flex-shrink).
    • C'est comme dire : "Dans un monde idéal, j'aimerais faire 200px de large. Mais je sais qu'on est dans Flexbox, donc je m'adapterai s'il le faut !"
    • ⚠️ Sur l'axe principal (en ligne), flex-basis remplace la propriété width (largeur).

    📙 Super Raccourci : La Syntaxe flex

    La propriété flex combine les trois réglages : flex-grow, flex-shrink et flex-basis dans une seule formule et dans un ordre très précis. 
              /* L'ordre est toujours le même : Grow, Shrink, Basis */
          .ma-boite {
             flex: [flex-grow] [flex-shrink] [flex-basis];
          }
    Exemple : 
              .ma-boite {
            flex: 1 0 200px; 
          }
    Traduction pour le navigateur : "Ta taille idéale est de 200px (basis). S'il reste de la place, tu grandis pour la prendre (grow: 1). S'il manque de la place, tu refuses de rétrécir (shrink: 0)."

    Les "Cheat Codes": des versions encore plus courtes
    • flex: 1; : C'est le plus courant ! Le navigateur comprend automatiquement que tu veux dire « Prends toute la place disponible de manière égale avec tes voisins, peu importe ta taille de base ».
    • flex: initial; : C'est le comportement par défaut (« Reste à ta taille normale, ne grandis pas, mais rétrécis si besoin »).

    📙Kit de Survie FlexBox

    Je veux ranger des éléments ? → display: flex; sur le parent.
    Dans quel sens ? → flex-direction (ligne ou colonne).
    Alignement sur la ligne ? → justify-content .
    Alignement en hauteur ? → align-items .
    Besoin d'espace ? → gap .
    Qui prend l'espace en trop ? → flex-grow (Par défaut : 0, personne ne grandit).
    Qui se sacrifie quand il manque de l'espace ? → flex-shrink (Par défaut : 1, tout le monde se serre).

    CSS | Grid (La Grille)

    📙 La Différence entre Flexbox et Grid

    Flexbox = 1 Dimension.
    Il gère les éléments soit sur une ligne (horizontal), soit sur une colonne (vertical). C'est parfait pour aligner des boutons ou un menu.

    Grid = 2 Dimensions.
    Il gère les éléments sur les lignes ET les colonnes en même temps. C'est parfait pour construire la structure globale d'une page entière (l'en-tête en haut, le menu à gauche, le contenu au centre, le pied de page en bas).

    📙 Grid : La Méthode Mathématique :

    1. Créer la Grille (Sur le Parent)

      Comme pour Flexbox, tout commence sur le conteneur parent. Dès qu'on écrit display: grid;, on transforme notre boîte en une grille invisible.
      Ensuite, on doit dire au navigateur combien de colonnes et de lignes on veut, et quelle doit être leur taille. Pour cela, l'unité de mesure : le fr (Fraction).

      1fr signifie "une part du gâteau". Si on a trois colonnes de 1fr, l'espace est coupé en trois parts égales.

      Les 3 propriétés de base à noter pour le parent
      Propriété Ce qu'elle fait Exemple
      grid-template-columns t le nombre et la largeur des colonnes (verticales). 1fr 1fr 1fr (Crée 3 colonnes de taille égale)
      grid-template-rows Définit la hauteur des lignes (horizontales). 100px auto (Ligne 1 = 100px, Ligne 2 = s'adapte)
      gap L'espace vide entre les cases. 20px
    2. Placer les éléments (Sur les Enfants)

      Une fois que notre grille est dessinée, on peut dire à n'importe quelle boîte (les enfants) : "Toi, tu vas t'étirer de la colonne 1 à la colonne 3".

      ⚠️ Piège à éviter :
      Quand on place un élément, on ne compte pas les cases, on compte les lignes de séparation !

        Imaginons un mur avec 3 fenêtres. Pour encadrer ces 3 fenêtres, il y a 4 piliers. En Grid, on compte les piliers.
      • Ligne 1 = Le bord tout à gauche.
      • Ligne 2 = La première séparation.
      • Ligne 3 = La deuxième séparation.
      • Ligne 4 = Le bord tout à droite.

      Pour dire à une boîte de prendre les deux premières colonnes, on écrit : 

                    .ma-boite {
                grid-column: 1 / 3; /* Commence à la ligne 1, s'arrête avant la ligne 3 */
              }

    Aide-Mémoire Si on dois construire une page, 2 questions à se poser :
    Est-ce une structure globale ? (Mise en page, tableau de bord, galerie complexe) → Utiliser Grid.
    Est-ce un alignement local ? (Boutons d'un menu, icône à côté d'un texte) → Utilise Flexbox.

    📙 Grid : la Méthode 100% Visuelle

    1. Étape 1 : Donner un nom aux enfants
      Plutôt que de dire à une boîte "va de la ligne 1 à 3", on va simplement lui coller une étiquette sur le front grâce à la propriété 
                .mon-en-tete {
            grid-area: header;
          }

          .mon-menu {
            grid-area: sidebar;
          }

          .mon-contenu {
            grid-area: main;
          
          }
        .mon-pied-page {
          grid-area: footer;
        }
    2. Étape 2 : Dessiner la carte sur le parent
      Maintenant, on retourne sur notre conteneur principal (display: grid;). Avec la propriété grid-template-areas, on va littéralement "dessiner" notre mise en page en écrivant les noms des zones. Chaque mot représente une case. 
                .conteneur {
            display: grid;
            grid-template-areas:          /* On dessine une grille avec des mots entre guillemets */
              "header header header"
              "sidebar main main"
              "footer footer footer";
          }
        Le code ressemble bien à la page finale :
      • le header prend 3 cases en haut,
      • la sidebar prend 1 case à gauche,
      • le main prend 2 cases à droite,
      • et le footer prend 3 cases en bas.

    C'est génial pour la maintenance de notre code.
    Si demain notre client dit "Finalement, je veux le menu à droite plutôt qu'à gauche", on n'a pas besoin de recalculer toutes tes lignes. On a juste à inverser les mots "main sidebar" dans notre carte !
