C'est quoi cette page ?

C'est mon bac à sable.

Utiliser un bac à sable (ou sandbox en anglais) est l'une des méthodes les plus efficaces pour progresser, et en particulier en développement web.

    1. La métaphore du bac à sable : Liberté et Sécurité
    • Liberté totale : Je peux construire un château magnifique, puis l'écraser deux minutes après pour pouvoir recommencer autre chose.
    • Sécurité : Si le château s'écroule, personne ne se blesse. Ce qui se passe dans le bac à sable n'impacte pas le reste du monde.
  2. En informatique : Un environnement isolé
    Dans mon contexte de création de page web, le bac à sable est un environnement de test isolé. C'est un espace où je peux exécuter, modifier, tester du code
    Ce que cela permet de faire :
  • L'expérimentation pure : "Que se passe-t-il si je mets ce display: grid ici ?"
  • Le droit à l'erreur : Je peux tester des scripts complexes. Si ça ne marche pas ou que l'affichage devient illisible, ce n'est pas grave : c'est fait pour ça.
  • L'apprentissage par la pratique : C'est le pont entre la théorie des cours et la réalité du code.
    Pourquoi c'est crucial pour toi ?
  • En créant cette page web "bac à sable", je développe la mémoire musculaire du code.
  • C'est un espace où je suis le seul maître à bord : je m'amuse avec les nouvelles techniques que j'apprends, et c'est très satisfaisant de les voir s'afficher ou s'animer comme je l'avais visualiser dans ma tête.
